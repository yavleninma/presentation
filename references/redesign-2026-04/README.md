# redesign-2026-04

Этот каталог хранит новый HTML-референс редизайна как исходный файл, доступный внутри репозитория.

## Что здесь главное

- `preview.html` — неизменённый исходник, от которого идёт текущая рокировка интерфейса.
- Для UI-работ по этому редизайну сначала читать именно его, а уже потом сверять проектные документы.

## Как читать `preview.html`

- Верхний masthead, блоки `Компоненты MVP`, `Что задаёт шаблон, а что задаёт цвет`, `Состояния`, `Не делать` и кодовый блок — это документационный слой референса, а не прямой пользовательский интерфейс.
- Подписи `Экран 1`, `Экран 2`, `Экран 3`, `Экран 4` и `Скрытый слой` нужны агенту для навигации по референсу. В продукт их переносить не нужно.
- Из HTML в продукт извлекаются только 4 видимых состояния:
  - тихий вход;
  - короткий мини-чат;
  - редактор с закрытым drawer;
  - редактор с открытым drawer.
- `fit-pass` остаётся скрытым служебным проходом после генерации. Это не отдельный экран для пользователя.

## Что нельзя переносить в UI как есть

- Самоописание продукта: как он устроен, что в него входит, чего в нём нет, почему он хороший.
- Служебные подписи и навигацию по референсу: `Экран N`, `Скрытый слой`, `Pipeline`, `служебный слой`, `hidden by default`.
- Объяснения интерфейса самому себе: `без stepper`, `без промо-героя`, `left rail`, `center canvas`, `right drawer closed`, `локальная правая панель`.
- Внутренние или технические комментарии про `fit-pass`, `MVP`, `канон`, `продукт`, `систему`.

## Правило конфликта

Если старые дизайн-доки проекта спорят с этим референсом, для текущего редизайна сначала опираемся на `preview.html`, а конфликт потом снимаем обновлением документов.
