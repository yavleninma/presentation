# UI_KIT_V0.md

Рабочая опора для UI прототипа `presentations-frontend`.
Не полная дизайн-система — минимум, достаточный, чтобы агент не плавал по вкусу.

Приоритет при конфликтах: **Figma v3 — Brand Kit** и **`DESIGN_DECISIONS.md`** § v3 → этот файл → `DESIGN_SPINE.md`.
Маршрут и схемы экранов — в `SCREEN_ATLAS.md`. Правила текстов — в `COPY_RULES.md`.

---

## Характер

Светлый, строгий, спокойный. Без стартапной пестроты, без банковской мертвечины.
Центр внимания — слайд, не оболочка. Оболочка помогает, не доминирует.

## Типографика

Шрифт: **Inter** (Bold / SemiBold / Medium / Regular). Единый на все экраны.
Моноширинный слой (debug, технические строки): **IBM Plex Mono** с подмножеством `cyrillic`.

Правила:
- Заголовки экранов — максимум 5–7 слов.
- Подзаголовки — только если без них непонятно.
- Метки разделов, счётчики, вторичные подписи — `--text-secondary`, меньший размер.
- Тексты на слайдах звучат как настоящая презентация для руководителя.
- Запрещённый словарь — в `COPY_RULES.md`.

## Компоненты оболочки

### Composer (Старт)
Белое поле с `--radius-start-field`, тень `--shadow-start-composer`.
Кнопка «↑» — синяя капсула `--accent-primary`, `--radius-pill`, справа от поля.
При многострочном вводе кнопка прижата к нижнему краю (`align-self: flex-end`).

### Scenario Chip
Фон `--start-chip-bg` (#F1F3F9), текст `--start-chip-text`, радиус `--radius-sm`.
При наведении — лёгкий darkening без синего. Не suggestion chip.

### Chat Card
Максимальная ширина `--chat-card-max-width` (47.5rem / 760px), фон `--bg-card`.
Тень `--shadow-chat-card`, радиус `--radius-md` (20px).
Внутри: шапка → сепаратор → лента сообщений → [сепаратор → compose].

### Chat Bubble
- **Бот «В»**: аватарка `--accent-primary` (#2D4BEE); баббл фон `--bubble-bot`.
- **Пользователь «М»**: аватарка `--accent-coral` (#F26161); баббл фон `--bubble-user`.
- Радиус: `--radius-bubble-main` / `--radius-bubble-tail`.

### Suggestion Chip
Обводка `--accent-primary`, текст `--accent-primary`, фон прозрачный.
Радиус `--radius-pill`. При клике — сразу отправляется как ответ.

### Chat CTA
«Собрать черновик →» — полная ширина блока, фон `--accent-primary`, текст белый, `--radius-pill`.
Появляется только когда `session.readyToBuild = true`.

## Компоненты черновика (Draft screen)

### Slide Text Card
Карточка текстового содержимого одного слайда: заголовок, подзаголовок, буллеты.
Фон `--bg-card`, радиус 12px, тень `--shadow-card`. Поля редактируемые inline.
В состоянии загрузки показывает скелетон (skeleton placeholder).

### Draft Chat (боковая панель)
Узкая колонка справа: шапка «Помощник», лента бабблов, compose-поле.
Та же баббл-система, что в мини-чате.

## Компоненты редактора

### Topbar
Высота 56px, фон `--bg-card` / `--bg-shell`, нижняя граница `--border-soft`.
Слева: `←` (ghost, ArrowLeft) + brand mark + название документа (редактируемое inline).
Справа: `[Шаблон ▾]` + `[Цвет ▾]` — ghost-селекторы, радиус `--radius-xs`; `✦` (Sparkles, ghost-иконка).

### Rail (лента миниатюр)
Ширина 160px, фон `--bg-card-soft`, правая граница `--border-soft`.
Активная миниатюра — обводка `--accent-primary` (2px).

### Canvas-зона
Фон `--bg-canvas` (#EDF2F7). Слайд 16:9 по центру — белый, `--radius-lg` (24px).
Accent bar `--accent-success` (#19A463) вверху слайда, высота 4px.

### Drawer (панель настроек слайда)
Ширина 280px, фон `--bg-card`, левая граница `--border-soft`.
Заголовок «Настройки слайда» + кнопка «×» (X, ghost).
Секция AI-действий (Упростить / Добавить цифры / Сократить) — ghost-кнопки на всю ширину.
Секция «Заметки» — поле `--bg-card-soft`, без жёсткой границы.
Открывается кнопкой Sparkles в topbar. Текстового лейбла «AI» рядом с иконкой — нет.

### Footer редактора
Фон `--bg-card-soft`, верхняя граница `--border-soft`.
Содержит: `ChevronLeft` — «Слайд N из M» — `ChevronRight`.

## Компоненты слайда (внутри Canvas)

Эти компоненты живут внутри слайда, а не в оболочке.

### Section Label
Маленькая верхняя метка раздела слайда. Текст `--text-secondary`, маленький размер, разрядка.

### KPI Card
Одна метрика: иконка + название + число + change badge (рост/падение/стабильно).

### Content Card
Блок содержимого: заголовок + иконка + 2–4 строки смысла.

### Risk Card
Риск: формулировка + влияние + мера + флаг «нужен шаг сверху».
Левая полоска — `--accent-warning` или `--accent-success`.

### Ask Card
Запрос к руководителю: что нужно + почему сейчас + что будет, если не решить.

### Placeholder Chip
Мягкая подсказка внутри слайда о недостающих данных.
Фон `--accent-warning-soft`, текст `--accent-warning`, радиус `--radius-sm`.

## Голос и тон

- Продукт говорит кратко, по делу.
- Не объясняет себя. Не продаёт себя. Не оправдывается.
- Заголовки экранов — максимум 5–7 слов. Не предложение, не абзац.
- Подзаголовки — только если без них реально непонятно.
- Запрещённый внутренний словарь и полные правила — в `COPY_RULES.md`.

---

## Антипаттерны

- Иконки из разных библиотек (только `lucide-react`).
- Тяжёлая диагностическая панель вместо слайда.
- Input-heavy экран перед первым результатом.
- Счётчик символов / прогресс-бар ввода как главный элемент.
- Разные визуальные языки на соседних экранах.
- Много вторичных кнопок одновременно.
- Объяснительные параграфы о том, что делает продукт.

---

## Правило вкуса

Если элемент не делает артефакт яснее, не усиливает ощущение собранности и не помогает пройти дальше — его не добавляют.
