# KIMI UX Playbook — Генерация как перформанс

> **Зачем этот файл:** Эталонный разбор UX-паттернов KIMI Agentic Slides, которые создают ощущение "умного соавтора + крутого дизайнера". Используется как скилл: стратег сверяет приоритеты, инженер подглядывает при реализации каждой фичи.
>
> **Источник:** Анализ KIMI (kimi.com/slides) на апрель 2026 — веб-версия, режимы Visual и Adaptive, агент OK Computer, модели K2.5 + Nano Banana Pro.
>
> **Принцип:** Не копируем интерфейс. Копируем **ощущение** и адаптируем под наш стек и рынок.

---

## 1. Главная идея: генерация — это ритуал, а не кнопка

У KIMI генерация презентации — не "нажал → подождал → получил". Это **управляемый поэтапный процесс**, где пользователь чувствует себя режиссёром, а AI — талантливой командой, которая работает на него.

### Почему это работает психологически

| Паттерн | Что чувствует пользователь | Как KIMI это делает |
|---------|---------------------------|---------------------|
| **Прозрачность** | "Я вижу, что происходит — мне не тревожно" | Показывает этапы: исследование → план → дизайн → сборка |
| **Контроль** | "Я решаю, а не AI за меня" | Редактируемый аутлайн ДО генерации слайдов |
| **Скорость** | "Вау, быстрее чем я думал" | Первый видимый результат (аутлайн) за 10-15 секунд |
| **Качество** | "Это выглядит как от дизайнера" | AI-инфографика, единая палитра, профессиональная типографика |
| **Партнёрство** | "AI думает вместе со мной" | Добавляет слайды, которые ты не просил, но они нужны (SWOT, спикер-ноуты) |

### Ключевой инсайт для SlideForge

Сейчас у нас: `промпт → спиннер → готовая презентация`. Пользователь теряет контроль на 30-60 секунд — это **мёртвая зона**, где нарастает тревога ("что там? получится ли?"). KIMI разбивает эту зону на этапы, каждый из которых даёт дофамин: "о, план готов", "о, шаблон выбран", "о, слайд 3 уже собирается".

## 2. Пятиактная структура генерации KIMI

KIMI разбивает создание презентации на пять явных стадий. Каждая видна пользователю, каждая интерактивна.

### Акт 1: Ввод (Input Stage)

**Что делает KIMI:**
- Текстовое поле с промптом (главный путь)
- Загрузка файлов: PDF, DOCX, Excel, изображения — до 50 файлов, до 100 МБ
- Вставка URL — AI сам парсит контент страницы
- Выбор режима: Visual (быстрый, 5-10 мин) vs Adaptive (глубокий, 30-60 мин)
- Указание количества слайдов (вручную или "автоматически")
- Миниатюра шаблона (Freestyle по умолчанию — AI сам решает дизайн)

**Что создаёт ощущение:**
- "Я могу подать материал ЛЮБОГО формата — AI разберётся"
- Переключатель Visual/Adaptive = пользователь выбирает "сколько времени потратить"
- Freestyle-шаблон = "AI сам дизайнер, доверься ему"

**Что забрать в SlideForge:**
- [ ] Промпт-подсказки (6 кнопок-примеров) — уже в канбане (EPIC-13)
- [ ] Загрузка файлов — EPIC-07, но поднять приоритет как UX-фактор
- [ ] Режим "быстрый vs глубокий" — можно имитировать через переключатель "краткая / детальная" генерация (разное кол-во слайдов + разная детальность промпта)
- [ ] Freestyle (авто-дизайн) — у нас сейчас всегда выбор шаблона; добавить опцию "AI выберет сам"

### Акт 2: Исследование + Аутлайн (Research & Outline)

**Что делает KIMI:**
- AI показывает, что он "исследует тему" — видны шаги: поиск информации, анализ, структурирование
- Результат: **редактируемый аутлайн** — список слайдов с заголовками и ключевыми тезисами
- Пользователь может: добавить/удалить секции, переупорядочить, уточнить содержание
- Только после одобрения аутлайна → переход к генерации визуала

**Что создаёт ощущение:**
- "AI не просто форматирует мой текст — он ДУМАЕТ о структуре"
- "Я контролирую план до того, как будут потрачены ресурсы на дизайн"
- "Это как брифинг с дизайнером — сначала структура, потом красота"

**Что забрать в SlideForge:**
- [ ] **EPIC-06 (Outline Editor) — поднять в приоритет #1.** Это самый большой UX-разрыв между нами и KIMI
- [ ] Показывать аутлайн в реальном времени через SSE (у нас уже есть `onOutline` callback!)
- [ ] Аутлайн должен быть РЕДАКТИРУЕМЫМ: drag-and-drop порядка, удаление/добавление слайдов, редактирование заголовков
- [ ] Кнопка "Выглядит хорошо, генерируй!" — явный переход от плана к рендеру

### Акт 3: Выбор визуального стиля (Template Selection)

**Что делает KIMI:**
- Библиотека шаблонов разных типов: бизнес, образование, research, и т.д.
- Freestyle-режим: AI анализирует тему и сам подбирает стиль
- Выбор происходит ПОСЛЕ аутлайна, но ДО генерации слайдов
- Палитра цветов подстраивается автоматически под тему

**Что создаёт ощущение:**
- "Каждый этап — моё решение, а не автомат"
- "Шаблоны выглядят как премиум, а не как шаблоны Google Slides"
- "AI предлагает стиль, подходящий моей теме — он понимает контекст"

**Что забрать в SlideForge:**
- [ ] Перенести выбор шаблона МЕЖДУ аутлайном и генерацией (сейчас выбор шаблона — до промпта, что неинтуитивно)
- [ ] Увеличить библиотеку шаблонов (EPIC-14) — разнообразие = ощущение премиальности
- [ ] "Авто-шаблон" — AI предлагает шаблон на основе темы промпта (mapping: "финансы" → строгий, "стартап" → яркий)

### Акт 4: Генерация слайдов (Live Generation)

**Что делает KIMI:**
- Слайды собираются один за другим — видно, как каждый "появляется"
- AI генерирует контент (текст, тезисы) + визуал (инфографика, иллюстрации, диаграммы)
- Пользователь видит прогресс: "Слайд 3 из 10..."
- Nano Banana Pro генерирует уникальные инфографики под каждый слайд (не стоковые фото!)
- Брендовые цвета, единая палитра, профессиональная типографика — без участия человека
- Иногда AI добавляет слайды, которые не были в аутлайне (SWOT-анализ, рекомендации)

**Что создаёт ощущение:**
- "Я наблюдаю, как креативная команда работает на меня"
- "Каждый слайд — маленький вау-момент"
- "Это не шаблонная заливка текстом — это дизайн"

**Что забрать в SlideForge:**
- [ ] **Streaming слайдов по одному** — у нас уже есть! Но усилить: анимация появления каждого слайда, автоскролл к текущему
- [ ] **Статус-бар с этапами**: "Генерирую слайд 3 из 8... | Подбираю изображения... | Финализация"
- [ ] **Превью каждого слайда в реальном времени** — не ждать всю деку, показывать готовые слайды сразу
- [ ] **Инфографика** — пока у нас фото с Pexels. На будущее: AI-генерация визуалов (Kandinsky/DALL-E) для "дизайнерского" ощущения
- [ ] **"Умная переподготовка"** — AI иногда добавляет бонус-слайд (итоги, рекомендации). Это чит-код для wow-эффекта

### Акт 5: Полировка и экспорт (Edit & Export)

**Что делает KIMI:**
- Полноценный редактор прямо в браузере: drag-and-drop элементов, форматирование текста, слои
- Перегенерация отдельных блоков: "переписать текст", "перегенерировать картинку"
- Слайд-переходы (анимация между слайдами) — включаются при экспорте
- Экспорт: PPTX, Google Slides (прямой), PDF, JPG/PNG
- Режим презентации (slideshow) прямо в браузере

**Что создаёт ощущение:**
- "Это не черновик — это почти финальная версия"
- "Я могу подправить детали, а не переделывать с нуля"
- "Экспорт работает везде — я не привязан к сервису"

**Что забрать в SlideForge:**
- [ ] **Перегенерация на уровне блока/слайда** — кнопка "Переписать" для заголовка, буллетов, всего слайда
- [ ] **"Сделай короче / формальнее / проще"** — быстрые команды без полной регенерации
- [ ] **Презентационный режим** (slideshow) — сейчас не реализован
- [ ] Экспорт в PDF (EPIC-05)

## 3. Театр прогресса: как показывать работу AI

Самая важная разница между "просто генератором" и "умным соавтором" — **видимость работы AI**. KIMI активно показывает, что происходит за кулисами.

### Паттерн: Thinking Steps (шаги мышления)

KIMI K2 Thinking использует "interleaved thinking" — AI думает между каждым действием и показывает эти шаги пользователю:

```
🔍 Исследую тему "Искусственный интеллект в медицине"...
📊 Нашёл 12 источников, анализирую ключевые тезисы...
📝 Формирую структуру: 8 слайдов, 3 секции...
🎨 Подбираю визуальный стиль для медицинской тематики...
📄 Генерирую слайд 1: "Титульный — AI в медицинской диагностике"
📄 Генерирую слайд 2: "Обзор рынка — рост применения AI"
🖼️ Создаю инфографику: "Процент внедрения AI по отраслям медицины"
📄 Генерирую слайд 3...
...
✅ Презентация готова! 8 слайдов, 3 инфографики, 2 диаграммы
```

### Как реализовать в SlideForge

У нас уже есть SSE стрим с событиями. Нужно расширить протокол:

**Текущие SSE-события:** `outline`, `slide`, `done`, `error`

**Целевые SSE-события (KIMI-style):**

| Событие | Когда | Что показывать пользователю |
|---------|-------|----------------------------|
| `thinking` | AI получил промпт | "Анализирую запрос..." |
| `researching` | AI формирует тезисы | "Собираю ключевые идеи для презентации..." |
| `outline` | Аутлайн готов | Показать интерактивный план слайдов |
| `outline_approved` | Пользователь одобрил | "Отлично! Приступаю к генерации..." |
| `template_selected` | Шаблон выбран | "Применяю стиль «Минимализм»..." |
| `slide_start` | Начало генерации слайда | "Генерирую слайд 3 из 8: «Анализ рынка»..." |
| `slide` | Слайд готов | Показать слайд + анимация появления |
| `image_search` | Поиск изображения | "Подбираю фото для слайда 5..." |
| `polishing` | Финальная доводка | "Проверяю консистентность стиля..." |
| `done` | Всё готово | "Готово! 8 слайдов созданы" + confetti? |

### Важно: "Фейковый" прогресс — это ОК

Не все шаги должны быть настоящими. "Проверяю консистентность стиля..." может быть 500ms задержкой перед `done`. Это нормально — это **UX-театр**, который создаёт ощущение тщательности. KIMI делает то же самое.

Но: шаги, где пользователь ДЕЙСТВУЕТ (аутлайн, шаблон), должны быть настоящими точками контроля.

## 4. Дизайнерское ощущение: как KIMI делает слайды "красивыми"

### 4.1. AI-сгенерированные визуалы vs стоковые фото

**KIMI:** Nano Banana Pro генерирует уникальные инфографики, иллюстрации и диаграммы ПОД КАЖДЫЙ СЛАЙД. Это не подбор стоковых фото — это создание визуала, который точно соответствует контенту.

**SlideForge сейчас:** Pexels стоковые фото. Это уже лучше, чем ничего, но между "стоковое фото с медсестрой" и "кастомная инфографика про AI в диагностике" — пропасть.

**Что делать:**
- Краткосрочно (сейчас): улучшить подбор фото — более точные imageQuery, приоритет бизнес-фото
- Среднесрочно: AI-генерация иллюстраций через Kandinsky/DALL-E для ключевых слайдов
- Долгосрочно: генерация инфографик на основе данных (SVG диаграммы, графики)

### 4.2. Единая цветовая палитра

**KIMI:** Автоматически подбирает палитру под тему. "Финансы" → тёмно-синий + золото. "Стартап" → яркий + энергичный. Все слайды в одной палитре.

**SlideForge:** У нас это уже реализовано через `template.colors.*`. Усилить:
- [ ] "Авто-палитра" — AI подбирает шаблон на основе темы промпта
- [ ] Больше шаблонов → больше палитр → ощущение персонализации

### 4.3. Профессиональная типографика

**KIMI:** Разные шрифты для разных шаблонов. Display-шрифты для заголовков, body-шрифты для текста. Вариативные размеры и начертания.

**SlideForge:** Сейчас всё на Inter. Все три шаблона выглядят одинаково по типографике.

**Что делать:**
- [ ] Уникальные шрифтовые пары для каждого шаблона (EPIC-13: "Разные шрифты для шаблонов")
- [ ] Display-шрифт для заголовков + body-шрифт для текста
- [ ] Google Fonts с поддержкой кириллицы: Golos Text, Manrope, Montserrat, PT Sans/Serif, Rubik, Nunito

### 4.4. Разнообразие лейаутов

**KIMI:** Каждый слайд — уникальный лейаут. Не 10 одинаковых "заголовок + буллеты", а: титульный, график, сравнение, цитата, таймлайн, KPI, изображение. KIMI варьирует лейауты ВНУТРИ деки.

**SlideForge:** У нас 10 типов слайдов — это уже хорошо! Но промпт должен генерировать РАЗНООБРАЗНУЮ комбинацию. Нужно проверить: не генерирует ли AI 8 из 10 слайдов как `content`?

**Что делать:**
- [ ] В промпт добавить explicit constraint: "Используй минимум 5 разных типов лейаутов" и "Не повторяй один layout более 2 раз подряд"
- [ ] Автоматически добавлять KPI-слайд, если тема содержит числа/метрики
- [ ] Автоматически добавлять timeline, если тема содержит даты/этапы

## 5. Два режима: быстрый vs глубокий

### Как это устроено у KIMI

| Параметр | Visual (быстрый) | Adaptive (глубокий) |
|----------|-------------------|---------------------|
| Время | 5-10 минут | 30-60 минут |
| Что делает | Генерирует контент из промпта | + ресерч темы в интернете |
| Визуал | Шаблонный стиль | Кастомные инфографики |
| Кому подходит | "Мне нужна дека через 5 минут" | "Мне нужна killer-презентация" |

### Как адаптировать для SlideForge

У нас нет web-ресерча (и не нужен на MVP) — но можно дать два режима:

**Режим "Быстрый" (3-5 слайдов, 15-30 сек):**
- Только ключевые слайды: титул + 2-3 контентных + финал
- Минимальный промпт к AI
- Для: "мне нужна заготовка прямо сейчас"

**Режим "Полный" (6-10 слайдов, 40-90 сек):**
- Полный аутлайн с разнообразными лейаутами
- Детальный контент + поиск изображений
- Для: "мне нужна рабочая презентация"

Переключатель: простой тумблер "Экспресс / Детальная" рядом с кнопкой генерации.

## 6. Умная перегенерация: ключ к ощущению "соавторства"

### Почему это важно

В KIMI можно кликнуть на ЛЮБОЙ элемент и перегенерировать его. Это превращает AI из "одноразового генератора" в **постоянного собеседника**, с которым ты итеративно доводишь результат до идеала.

### Уровни перегенерации (от дешёвого к дорогому)

| Уровень | Что делает | Стоимость (токены) | Приоритет |
|---------|-----------|-------------------|-----------|
| **Текстовый блок** | Переписать один заголовок или буллет-поинт | ~100 токенов | 🔴 Высокий |
| **Стиль текста** | "Короче", "формальнее", "проще" | ~200 токенов | 🔴 Высокий |
| **Весь слайд** | Перегенерировать контент слайда с тем же layout | ~500 токенов | 🟡 Средний |
| **Изображение** | Поискать другое фото / перегенерировать | ~1 API call | 🟡 Средний |
| **Layout слайда** | Сменить тип (content → two-columns) | ~300 токенов | ✅ Уже есть |
| **Вся дека** | Полная перегенерация | ~5000 токенов | ✅ Уже есть |

### Что реализовать

**MVP (быстрый эффект):**
- [ ] Кнопка "🔄 Переписать" при наведении на заголовок → AI переписывает только этот текст
- [ ] Кнопка "🔄 Переписать слайд" → AI перегенерирует контент, сохраняя layout и шаблон
- [ ] 3 quick-action кнопки на каждом слайде: "Короче" | "Формальнее" | "Переписать"

**Позже:**
- [ ] Свободный промпт к отдельному слайду: "Добавь больше цифр" / "Сделай в стиле McKinsey"
- [ ] Перегенерация изображения: "Другое фото" или вставка своего URL

## 7. "Бонусный" контент: AI как proactive-партнёр

### Что делает KIMI

При тестировании KIMI на промпте "10 слайдов о AI в здравоохранении" система:
- Сгенерировала 14 слайдов (больше, чем просили)
- Добавила SWOT-анализ, который не был в запросе
- Добавила спикер-ноуты к каждому слайду
- Добавила слайд "Рекомендации" в конце

Это **proactive over-delivery** — AI даёт больше, чем просили. Это создаёт вау-эффект.

### Как адаптировать для SlideForge

Добавить в промпт к LLM:

1. **Спикер-ноуты** — генерировать `notes` для каждого слайда (поле уже есть в типе `Slide`!). Показать их в правой панели под слайдом.

2. **"Бонусный" слайд** — если тема содержит проблему, добавить слайд "Выводы" или "Рекомендации". Если тема содержит метрики, добавить KPI-слайд.

3. **Авто-section dividers** — если слайдов > 6, автоматически добавлять section-слайды между логическими блоками.

## 9. Антипаттерны: чего НЕ копировать у KIMI

| Что у KIMI | Почему не копировать | Наша альтернатива |
|------------|---------------------|-------------------|
| 30-60 мин на "глубокую" генерацию | Неприемлемо для нашей аудитории. Люди хотят результат за минуту | Максимум 90 сек на полную деку |
| Web-ресерч (поиск информации) | Требует инфраструктуру + увеличивает latency | Пользователь сам даёт контент; AI структурирует |
| 50 файлов на вход | Over-engineering для MVP | Один файл (TXT/PDF) → достаточно для v1 |
| Огромная библиотека шаблонов | Нужны дизайнерские ресурсы | 5-7 качественных шаблонов лучше, чем 50 посредственных |
| Полноценный Canvas-редактор | Месяцы разработки | Inline-editing (уже есть) + перегенерация блоков |
| OK Computer (полностью автономный агент) | Затратно по LLM-токенам | Управляемый поэтапный flow с точками контроля |

## 10. Дорожная карта: от "генератора" к "соавтору"

### Фаза 1: "Я вижу, что происходит" (1 неделя)

Цель: убить "мёртвую зону" генерации.

- Театр прогресса: статусные сообщения во время SSE-стрима
- Анимация появления слайдов (fade-in + auto-scroll)
- Прогресс-бар: "Слайд 3 из 8"
- Промпт-подсказки (6 кнопок)
- Спикер-ноуты (генерация + UI)

### Фаза 2: "Я контролирую план" (1-2 недели)

Цель: дать пользователю контроль ДО генерации.

- Outline Editor: показать план, дать редактировать
- Выбор шаблона ПОСЛЕ аутлайна
- "Быстрый / Полный" режим
- Разнообразие лейаутов (constraint в промпте)

### Фаза 3: "AI — мой партнёр" (1-2 недели)

Цель: итеративная доводка без полной перегенерации.

- Перегенерация заголовка / буллетов / слайда
- Quick-actions: "Короче" / "Формальнее" / "Больше деталей"
- Бонусный контент: авто-section-dividers, авто-выводы
- Уникальные шрифты per template

### Фаза 4: "Это выглядит профессионально" (2-4 недели)

Цель: дизайнерское качество.

- 2-3 новых шаблона (Pitch Deck, Консалтинг, IT)
- AI-подбор шаблона (Freestyle-режим)
- AI-генерация иллюстраций (Kandinsky / DALL-E)
- Slideshow-режим
- PDF экспорт

## 12. Глоссарий терминов

| Термин | Определение (в контексте SlideForge) |
|--------|---------------------------------------|
| **Театр прогресса** | UX-паттерн: показывать пользователю "шаги мышления" AI, даже если часть из них декоративные |
| **Outline-first** | Принцип: сначала показать структуру, дать отредактировать, потом генерировать визуал |
| **Перегенерация блока** | Возможность переписать один элемент слайда (заголовок, буллеты, картинку) без перегенерации всей деки |
| **Quick-actions** | Набор из 2-3 кнопок при наведении на текст: "Короче", "Формальнее", "Переписать" |
| **Proactive over-delivery** | AI добавляет полезный контент, который пользователь не просил (спикер-ноуты, SWOT, рекомендации) |
| **Freestyle-режим** | AI сам выбирает шаблон/стиль на основе темы промпта |
| **Agentic Slides** | Подход KIMI: AI не просто форматирует текст, а исследует, структурирует и дизайнит автономно |
