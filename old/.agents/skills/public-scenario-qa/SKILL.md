# Public Scenario QA

Use this skill for public no-auth browser QA.

## Workflow

1. Start or locate the app.
2. Test the chosen public scenario in a browser.
3. Check clarity, friction, trust, momentum, one main job, one main CTA, honest affordances, calm visual style.
4. Check desktop first; add mobile when landing clarity, CTA flow, or layout matters.
5. Report:
   - scenario tested
   - user goal
   - what worked
   - blockers
   - friction
   - trust issues
   - cosmetic roughness
   - remaining risks
   - product rules check

## Rules

- Public no-auth scope only.
- Do not fix code as part of QA.
- Do not claim browser coverage from code inspection.
