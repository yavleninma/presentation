# Public Scenario Reference

## Scenario 1 - First Impression On `/`

- Goal: understand the product in a few seconds
- Check: one main job, one main CTA, task-first copy, calm trustable UI

## Scenario 2 - Generate From A Work Topic On `/`

- Goal: go from work context to a draft deck
- Check: minimal friction, honest progress, no fake affordances

## Scenario 3 - Draft Workspace After Generation

- Goal: inspect, refine, and export without feeling lost
- Check: confidence, clarity, honest labels, safe-to-use feeling

## Scenario 4 - Demo Honesty On `/demo`

- Goal: inspect examples without being misled
- Check: demos feel like real examples, not fake proof

## Scenario 5 - Mobile Clarity

- Goal: understand and start the product on a small screen
- Check: CTA remains visible, layout stays readable, flow stays focused
