# SlideForge - Strategy

> Updated: 2026-04-03

## Positioning

SlideForge turns work context into a presentable internal slide deck fast.

Not a design toy. Not a fake enterprise wizard. A calm work tool that gets the user to a safe draft quickly.

## Product Rules

1. Start from the work task, not from product hype.
2. Optimize for "not embarrassing to show" before visual flourish.
3. Keep one main job and one main CTA above the fold.
4. Hide unbuilt features or label them honestly.
5. Prefer calm, corporate-safe, low-clutter UI.
6. Use placeholders when data is missing. Never invent numbers.
7. Add complexity only after the simple flow is trustworthy.

## Priorities

1. Align `/` and `/demo` with the product rules.
2. Tighten preview-to-export fidelity.
3. Keep outline and regeneration flows lightweight.
4. Keep advanced scenarios hidden until the simple public flow is solid.

## Risks

1. Marketing drift back into vague "AI presentation generator" language.
2. Chasing visual flash before trust and output quality.
3. Showing half-built features as if they work.
4. Using invented proof or numbers instead of placeholders.
