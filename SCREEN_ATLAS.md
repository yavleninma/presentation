# SCREEN_ATLAS.md

Схематичная карта экранов прототипа в `presentations-frontend`.
Нужна, чтобы агент и человек видели маршрут и распределение ролей без открытия кода.

Опора по визуалу — **Figma v3 — Brand Kit** и **`DESIGN_DECISIONS.md`** § v3.
Опора по маршруту — **этот документ** и фактическое поведение кода прототипа.
При конфликте между документами: по токенам и хрому — Figma v3; по маршруту и скрытым проходам — этот атлас.

Правила чтения:

- **ASCII-схемы** — структура и порядок элементов, не пиксели.
- **Тексты в схемах каноничны.** Если схема говорит «Собрать черновик →», в коде должно быть именно это, а не вольный пересказ. Улучшать формулировку можно, раздувать — нельзя. Все правила текстов — в `COPY_RULES.md`.
- Внутренние пометки вроде «Экран N» служат навигацией в документе и **не копируются в UI**.

---

## Маршрут прототипа

```
[Старт: idle]
      ↓ ввод запроса
[Старт: мини-чат]
      ↓ «Собрать черновик →»
[Черновик: сетка слайдов + чат]     ← screen = "draft"
      ↓ «Собрать черновик →»
[Редактор]                          ← screen = "editor"
      ↓ Sparkles
[Редактор + панель]
```

Скрытый служебный проход **fit-pass** выполняется после генерации и не является пользовательским экраном.

> **Расхождение с Figma v3:** в Figma v3 есть экран Pattern Selection (выбор паттерна «Питч / Отчёт / Обучение»). В прототипе этот экран **отсутствует**: структура определяется автоматически по запросу. Экран черновика (`draft`) в Figma v3 отсутствует — это добавление прототипа.

**Роль:** принять рабочий запрос.
**Код:** `PrototypeScreen = "start"`, `EntryPhase = "idle"`

```

│  ◼ Вняtно                                                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
```

Визуальный язык v3:

- Composer-карточка белая, тень `0 4px 12px rgba(21,28,46,0.08)`, радиус pill.

Не должно быть: счётчиков символов, валидации до отправки, параграфов о том, что делает продукт.

Не должно быть: прогресс-счётчика вопросов на видном месте (только `sr-only`), объяснений что делает чат.

## Что проверять на каждом экране

1. Ясно ли, где находится пользователь?
2. Ясно ли, что делает система?
3. Ясно ли, что будет следующим шагом?
4. Показываем ли мы объект как можно раньше?
5. Нет ли диагностического шума вместо результата?
