# Presentation

Репозиторий перезапущен под текущую версию Vnyatno.

## Текущий маршрут

Сейчас в продукте один рабочий путь:

`start -> mini-chat -> build -> editor`

Пользователь начинает с тихого входа, коротко уточняет смысл в mini-chat, затем получает собранный черновик и попадает в editor для точечной локальной пересборки активного слайда.

## Что это умеет сейчас

Текущая версия в `presentations-frontend/` умеет:

- принять рабочий запрос без раннего выбора темы и формата;
- быстро добрать недостающий контекст в mini-chat;
- собрать шестислайдовый draft;
- показать editor с левой лентой, холстом и закрытым drawer по умолчанию;
- локально пересобрать только активный слайд;
- честно оставить пустоты там, где фактуры пока нет.

## Что пока не обещаем

- Нет большого onboarding.
- Нет отдельного экрана `понимание`.
- Нет отдельного экрана `план`.
- Нет экспорта.
- Нет серверной генерации через модель.
- Нет расширенного набора сценариев поверх текущих рабочих кейсов.

## Проверка

Базовый локальный проход:

```bash
npm run verify
```

Отдельный smoke для маршрута и интерфейса живёт в `.ux-audit/ux-audit.spec.cjs`. Это локальная проверка, не CI-gate.

Для production-сборки используется тот же workspace:

```bash
npm run start --workspace presentations-frontend
```

## Git и деплой

Для репозитория действует одна рабочая ветка `main`.
Коммит и пуш делаем только по явной команде пользователя.

Для Vercel опорой считается `presentations-frontend` как root directory.
Автодеплой через Git-интеграцию Vercel отключён в `presentations-frontend/vercel.json`.
Единственная production-дорога идёт через `.github/workflows/presentations-ci.yml`: push в `main` проходит `npm run verify`, затем `vercel deploy --prod`, затем перепривязку `vnyatno.vercel.app` к свежему deployment.

Целевой production-адрес:

`https://vnyatno.vercel.app`
